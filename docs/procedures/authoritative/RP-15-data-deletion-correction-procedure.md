# RP-15 — Data Deletion & Correction Procedure

## Trigger
User, operator veya compliance workflow veri silme veya düzeltme talebi üretiyor.

## Roles
- request owner
- privacy review owner
- data surface owner

## Steps
1. Request subject ve scope'u doğrula.
2. Affected data classes ve systems listesini çıkar.
3. Eligibility ve blocking legal constraints varsa kaydet.
4. Deletion veya correction action planını oluştur.
5. Completion evidence ve requester outcome bilgisini kaydet.

## Default Timelines
- valid request acknowledged in next review window
- blocked requests documented reason taşımalı

## Completion
- completed
- partially_completed_with_reason
- denied_with_reason
